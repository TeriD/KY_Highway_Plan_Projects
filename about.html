Abstract

The Active Highway Projects Viewer is an open-source, public-facing interactive dashboard that enables users to explore current and awarded highway construction projects in Kentucky — independently of the Esri software stack. Built with modern JavaScript frameworks and leveraging open data APIs and spatial visualizations, the application empowers the public, contractors, and government stakeholders with intuitive access to authoritative project data.

At its core, this tool visualizes the Approved Six-Year Highway Plan (6YP)—a legislatively approved framework updated every two years by the Kentucky General Assembly. The 6YP outlines major highway improvement projects across the state and is developed through collaborative input from KYTC, Area Development Districts (ADDs), Metropolitan Planning Organizations (MPOs), and Highway District Offices. The Viewer displays currently active and awarded projects derived from this plan, with data reflecting the availability of both state and federal highway funds.

Designed for user experience, accessibility, performance and responsiveness, the dashboard supports interactive filtering, dynamic charting, map-based exploration, and integration with state-maintained APIs, making it a valuable tool for planning, transparency, and public engagement in infrastructure development.

Problem Statement

The Kentucky Transportation Cabinet (KYTC) offers GIS-based access to construction project data using Esri proprietary tools. However, this approach limits public access due to licensing, performance issues, and limited interoperability. Users looking to explore highway projects often find the existing interfaces unintuitive and inflexible.

This project aims to create an open-source, device-responsive, non-ESRI dashboard that presents current transportation project data in a visually informative and user-friendly manner. It also aims to develop modular functions for integration into similar dashboards.

Project Design and Implementation

Goals and Objectives

The project initial goals and objectives included in the original project proposal have been implemented in current design as follows::

  1. Interactive dashboard using open web technologies to explore active KYTC highway projects

    ✔️ Implemented. The dashboard uses HTML, CSS, and JavaScript for interactive data exploration.

  2. Replace reliance on proprietary mapping tools with Leaflet.js and open APIs

    3. ✔️ Implemented. Leaflet.js is used for mapping (see index.html), replacing Esri tools.

  3. Integrate public API endpoints, such as the KYTC Spatial API, to retrieve route-specific data.

    4. ✔️ Implemented. The dashboard includes integration with KYTC Spatial API for route data.

  4. Use SQLite to store and query project data for portability and offline access.

    5. ✔️ Implemented. SQLite is used to store project data, allowing for efficient querying and offline access. Modifications were made in the planned storage of the spatial data in geojson format for performance optimization in the map. While geojson strings can be stored in a TEXT column inside an SQLITE database, it was determined that the most direct method was to read directly from the geojson files.

  5. Provide a responsive, user-friendly interface for exploring project data on various devices.

    6. ✔️ Implemented. The dashboard is designed to be responsive and user-friendly, adapting to different screen sizes.

  6. Enable data export in common formats (CSV, Excel) for further analysis.

    7. ✔️ Implemented. The dashboard allows users to export project data in CSV and Excel formats directly from the integrated data table.

    Capstone Project Overview

    This project is a culmination of the skills and knowledge acquired throughout the CodeYou program. It demonstrates the ability to design and implement a complex, interactive web application that meets real-world needs. The project showcases the use of modern web technologies, data visualization techniques, and API integration to create a functional and user-friendly dashboard for exploring highway projects in Kentucky.

    The dashboard is built with open web technologies (HTML, CSS, JavaScript, Leaflet.js, Chart.js, and Tabulator.js), eliminating reliance on proprietary Esri tools. This removes licensing barriers and makes the tool freely accessible to the public.

    The interface is designed to be responsive, adapting to various devices and screen sizes. Interactive features like filtering, charting, and map-based exploration make the dashboard intuitive and easy to use, overcoming the inflexibility of previous solutions.

    By using lightweight libraries and direct GeoJSON data, the dashboard improves performance and ensures compatibility with other open data sources and APIs. SQLite is used for efficient data storage and querying, supporting offline access.

    Integration with the KYTC Spatial API and public data files allows users to explore route-specific project metadata without restrictions, supporting transparency and public engagement.

    Code Functionality and Use:

    JavaScript is the backbone of the dashboard's interactivity, visulization, and modularity, making the application functional, user-friendly, and responsive.

    It is essential for enabling all of the dynamic features of the dashboard:

    • Data Loading & API Integration:

      • JavaScript fetches project data from GeoJSON files, SQLite databases, and the KYTC Spatial API. This allows the dashboard to display up-to-date information and route-specific metadata.

    • Map Visualization:

      • Using Leaflet.js, JavaScript renders interactive maps, displays project locations, and enables user interactions such as clicking on features to view details.

    • Charting & Data Visualization:

      • JChart.js is used via JavaScript to generate dynamic pie and bar charts, helping users visualize project distributions and summaries.

    • Filtering & Sorting:

      • JavaScript powers the filtering logic, allowing users to select counties, districts, or project types. It updates the map, charts, and data table in real time based on user selections.

    • Interactive Data Table:

      • Tabulator.js is used to create sortable, filterable, and exportable tables. JavaScript manages the table’s data, updates it when filters change, and enables export to CSV/Excel.

    • Responsive UI & Modular Components:

      • JavaScript ensures the dashboard adapts to different devices and screen sizes, and supports modular architecture so components (map, charts, table) can be reused or extended.

    Modularization

    A challenging aspect of this project has been modularizing the dashboard's codebase. Ensuring that each component is self-contained and reusable requires careful planning and implementation.

    Modularizing the dashboard requires decoupling logic, managing shared state, abstracting third-party integrations, refactoring styles, and designing clear interfaces for each component. This can be time-consuming but will greatly improve reusability and maintainability.

    Some logic is intertwined between the map charts and table, making it challenging to separate them into distinct components. However, the overall architecture is designed to allow for future enhancements and modularization.
